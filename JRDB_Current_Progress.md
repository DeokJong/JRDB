# JRDB 프로젝트 필수 기능 정리 및 진행 사항

## 1. 프로젝트 개요

- **목표:** C++로 관계형 데이터베이스(RDB)를 개발하고, Java 애플리케이션이 JDBC를 통해 접근 가능하도록 구현
- **핵심 기능:** SQL 실행 엔진, 저장 엔진, 트랜잭션 관리, 동시성 제어, 인덱스 지원, 장애 복구 기능 제공

---

## 2. 필수 기능 목록 및 진행 상황

### ✅ **1. 저장 엔진 (Storage Engine)**

| 기능                 | 상세 설명                                             | 진행 상태 |
| ------------------ | ------------------------------------------------- | ----- |
| 테이블 생성 및 삭제        | 사용자가 테이블을 생성 및 삭제할 수 있도록 구현                       | ❌ 미진행 |
| 레코드 삽입, 조회, 수정, 삭제 | 기본적인 CRUD 연산을 지원 (INSERT, SELECT, UPDATE, DELETE) | ❌ 미진행 |
| 데이터 저장 구조          | 데이터 블록을 디스크에 효율적으로 저장하는 파일 시스템 구현                 | ❌ 미진행 |
| 페이지 및 버퍼 관리        | 페이지 교체 알고리즘 적용 (LRU, Clock 등)                            | ❌ 미진행 |

---

## 3. 시스템 아키텍처 설계

### ✅ **아키텍처 개요**

- **저장 엔진 (Storage Engine):** 데이터 저장 및 페이지 관리
- **SQL 실행 엔진 (Query Processor):** SQL 파싱 및 실행 계획 수립
- **트랜잭션 관리자 (Transaction Manager):** WAL 기반 트랜잭션 및 동시성 제어
- **JDBC 네트워크 모듈 (JDBC Connector):** Java 애플리케이션과 통신

---

### ✅ **모듈별 역할**

| 모듈 | 주요 기능 |
|------|----------|
| Storage Engine | 데이터 저장, 페이지 관리, 버퍼 관리 |
| Query Processor | SQL 파싱, 실행 계획 수립 및 최적화 |
| Transaction Manager | ACID 보장, WAL 기반 복구, 동시성 제어 |
| JDBC Connector | Java JDBC 연동, 네트워크 통신 |

---

### ✅ **데이터 흐름**

1. Java 애플리케이션이 JDBC를 통해 SQL 요청을 보냄.
2. Query Processor에서 SQL을 파싱하고 실행 계획을 생성.
3. Storage Engine이 데이터를 조회하거나 변경.
4. Transaction Manager가 트랜잭션을 관리 (ACID 보장).
5. 결과를 JDBC를 통해 응답.

## 5. 향후 계획

1. **저장 엔진 개발 시작** → 테이블 생성, 파일 시스템 설계부터 진행
2. **SQL 실행 엔진 기초 구축** → 파서 및 AST 변환 구현
3. **트랜잭션 관리 고도화** → WAL 기반의 Undo/Redo 구현 및 동시성 제어 적용
4. **쿼리 최적화** → 실행 계획 및 인덱스 활용 개선
5. **JDBC 연계 개발** → Java 애플리케이션과 DB 연결 지원
